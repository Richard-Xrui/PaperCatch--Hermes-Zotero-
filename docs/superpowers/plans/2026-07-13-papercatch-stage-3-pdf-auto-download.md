# PaperCatch Stage 3 + PDF Auto Download Implementation Plan

日期：2026-07-13
状态：阶段性完成；PC-005 至 PC-009、PC-014、PC-015、PC-019 已验证，OA PDF 核心/API/UI 已完成
范围：PC-005 至 PC-009、PC-014、PC-015、PC-019 的收口依赖，以及 PDF 自动下载链路

## 目标

把 PaperCatch 从“只产出 `pdf_url`”推进到“可隔离、可回退、可重试的本地 PDF 下载”。

## 非目标

- 不做 Zotero 附件写入
- 不做 OCR / 解析 / 文本抽取
- 不绕过访问控制、登录态或付费墙
- 不新增未复现的 PC 编号
- 不另建问题台账；状态和验证统一回写 `docs/ISSUE_FIX_LOG.md`
- 不调用真实 arXiv / Zotero / SMTP / LLM

## 依赖顺序

1. 先收口抓取链路：PC-007、PC-008，保证搜索参数真实传递且默认值不被悄悄覆盖。已完成并验证。
2. 再修复抓取结果稳定性：PC-005、PC-006、PC-009，保证失败传播、抓取 ID 落盘与 `run_status.json` 一致。
3. 再修复每日流水线：PC-014、PC-015，保证 merge / enrich 失败不被吞掉。
4. 最后收口导入边界：PC-019，保证 `cron_wrapper.py` 不再导入即执行副作用。

PDF 下载阶段应依赖以上链路都已稳定，否则下载成功与否无法被可靠调度和回滚。

## PDF 下载目标

- 对已合并且可公开获取的论文下载 PDF
- 下载结果落到本地可复用目录
- 与论文记录建立稳定映射
- 允许重复执行而不重复下载

## 存储约定

- 建议目录：`papers/pdfs/`
- 建议命名：`papers/pdfs/<arxiv_id>.pdf`
- 临时文件：同目录 `.part`
- 推荐字段：
  - `pdf_url`
  - `pdf_path`
  - `pdf_status`
  - `pdf_size_bytes`
  - `pdf_downloaded_at`
  - `pdf_attempts`
  - `pdf_last_error`
  - `pdf_last_checked_at`

## 原子落盘

- 先写 `*.part`
- 仅在完整校验通过后 `os.replace()` 覆盖正式文件
- 临时文件与目标文件必须同目录
- 失败时保留原文件，不留下半截 PDF

## 去重规则

- 以规范化后的 arXiv ID 作为主键
- 默认忽略版本号差异，只保留同一论文的唯一本地 PDF
- 若已存在且大小有效，直接跳过

## 断点与回退

- 优先支持 HTTP `Range`
- 若服务端不支持 Range，自动整文件回退
- 若断点续传内容不连续或校验失败，回退为整文件重试
- 下载中断后允许下次从 `.part` 恢复或重建

## 校验

- 必须检查 HTTP 状态码
- 必须检查 `Content-Type` 是否为 PDF 或可接受的二进制下载响应
- 必须校验 PDF 魔数 `%PDF-`
- 必须校验下载大小大于最小阈值
- 必须在完整落盘后再更新论文字段

## 失败分类与重试

- 临时失败：超时、连接重置、5xx、被限流
- 终止失败：404、403、明显非 PDF 内容、校验失败
- 对临时失败做有界重试和退避
- 对终止失败停止重试并记录明确错误

## 访问边界

- 仅下载公开可访问的 arXiv PDF
- 不通过代理、cookie、登录态或自动化手段绕过访问控制
- 遇到受限内容仅记录失败原因，不做规避

## Zotero 边界

- 本阶段不写 Zotero 附件
- Zotero 附件策略单独作为后续阶段

## 隔离测试矩阵

- 目录创建与命名
- `*.part` 原子替换
- 同一 arXiv ID 重复下载跳过
- `Range` 成功续传
- `Range` 不可用时整文件回退
- 非 PDF 响应拒绝
- 404 / 403 / 5xx 分类
- 有界重试与最终失败
- 并发重复触发下不生成重复文件
- 只用 temporary data，不碰真实服务

- 任何目录、字段或重试策略变更都必须先通过隔离测试
- 若校验或重试行为不稳定，优先回退为只读计划，不推进写入逻辑
